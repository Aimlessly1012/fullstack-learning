## 🧠 shadcn/ui 是什么？

shadcn/ui **不是**传统的组件库（不是 npm 包）。

**传统组件库（如 Ant Design）：**
```
你的项目 → import Button from 'antd'
           → 组件代码在 node_modules，你不能改
```

**shadcn/ui：**
```
pnpm dlx shadcn@latest add button
→ 把 Button 的源码复制到 src/components/ui/button.tsx
→ 代码在你的项目里，你可以随意修改
```

**优点：**
- 完全掌控代码，可以随意定制
- 不依赖第三方更新
- 样式基于 Tailwind CSS，灵活可扩展

**缺点：**
- 组件更新需要手动同步
- 初始代码量较多

## 安装 shadcn/ui

```bash
# 初始化（选 Radix → Nova 预设）
pnpm dlx shadcn@latest init

# 安装所需组件（一次性安装）
pnpm dlx shadcn@latest add button input label card table badge avatar \
  dropdown-menu sheet sidebar tooltip separator dialog select checkbox
```

安装后会在 `src/components/ui/` 目录下生成各组件文件。

## 🤔 路由分组是什么？

Next.js App Router 支持**路由分组**：用括号包裹的目录名不会出现在 URL 中。

```
src/app/
├── (auth)/                    ← 路由分组，URL 不含这层
│   ├── layout.tsx             ← 登录页布局（简单，无侧边栏）
│   └── login/
│       └── page.tsx           → 访问 /login
│
└── (dashboard)/               ← 路由分组，URL 不含这层
    ├── layout.tsx             ← 后台布局（含侧边栏）
    ├── dashboard/
    │   └── page.tsx           → 访问 /dashboard
    ├── users/
    │   └── page.tsx           → 访问 /users
    └── roles/
        └── page.tsx           → 访问 /roles
```

**好处：** `/login` 和 `/dashboard` 使用不同的 layout（布局），但 URL 依然简洁。

## 登录页

`src/app/(auth)/login/page.tsx`

```tsx
'use client'  // 客户端组件，因为需要 useState、事件处理

import { useState } from 'react'
import { useRouter } from 'next/navigation'
import { Button } from '@/components/ui/button'
import { Input } from '@/components/ui/input'
import { Label } from '@/components/ui/label'
import { Card, CardContent, CardDescription, CardHeader, CardTitle } from '@/components/ui/card'
import { Shield, Github, Loader2 } from 'lucide-react'

export default function LoginPage() {
  const router = useRouter()
  const [loading, setLoading] = useState(false)
  const [error, setError] = useState('')
  // 用对象管理表单状态，比多个 useState 更整洁
  const [form, setForm] = useState({ email: '', password: '' })

  async function handleLogin(e: React.FormEvent) {
    e.preventDefault()  // 阻止表单默认提交行为
    setLoading(true)
    setError('')

    try {
      const res = await fetch('/api/auth/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(form),
      })

      const data = await res.json()

      if (!res.ok) {
        setError(data.error)
        return
      }

      // 登录成功，跳转到仪表盘
      router.push('/dashboard')
      router.refresh()  // 刷新 Server Component 缓存
    } catch {
      setError('网络错误，请重试')
    } finally {
      setLoading(false)  // 无论成功失败，都关闭 loading
    }
  }

  return (
    <div className="min-h-screen flex items-center justify-center bg-muted/30">
      <Card className="w-full max-w-md">
        <CardHeader className="text-center">
          <div className="flex justify-center mb-2">
            <Shield className="h-10 w-10 text-primary" />
          </div>
          <CardTitle className="text-2xl">RBAC 权限管理系统</CardTitle>
          <CardDescription>登录你的账号</CardDescription>
        </CardHeader>

        <CardContent className="space-y-4">
          <form onSubmit={handleLogin} className="space-y-4">
            <div className="space-y-2">
              <Label htmlFor="email">邮箱</Label>
              <Input
                id="email"
                type="email"
                placeholder="admin@admin.com"
                value={form.email}
                onChange={e => setForm(f => ({ ...f, email: e.target.value }))}
                required
              />
            </div>

            <div className="space-y-2">
              <Label htmlFor="password">密码</Label>
              <Input
                id="password"
                type="password"
                placeholder="••••••••"
                value={form.password}
                onChange={e => setForm(f => ({ ...f, password: e.target.value }))}
                required
              />
            </div>

            {/* 错误提示 */}
            {error && (
              <p className="text-sm text-destructive">{error}</p>
            )}

            <Button type="submit" className="w-full" disabled={loading}>
              {loading && <Loader2 className="mr-2 h-4 w-4 animate-spin" />}
              登录
            </Button>
          </form>

          {/* 分隔线 */}
          <div className="relative">
            <div className="absolute inset-0 flex items-center">
              <span className="w-full border-t" />
            </div>
            <div className="relative flex justify-center text-xs uppercase">
              <span className="bg-background px-2 text-muted-foreground">或</span>
            </div>
          </div>

          {/* GitHub 登录 */}
          <Button
            variant="outline"
            className="w-full"
            onClick={() => router.push('/api/auth/github')}
          >
            <Github className="mr-2 h-4 w-4" />
            GitHub 登录
          </Button>
        </CardContent>
      </Card>
    </div>
  )
}
```

## 🧠 核心概念：Server Component vs Client Component

Next.js App Router 中，组件分两类：

| | Server Component | Client Component |
|--|--|--|
| 默认 | ✅ 是 | 需要 `'use client'` |
| 可以 fetch | ✅ 直接访问数据库 | ❌ 需要通过 API |
| 可以用 useState | ❌ | ✅ |
| 可以用事件处理 | ❌ | ✅ |
| SEO 友好 | ✅ | 需要额外配置 |

**规则：** 只在需要交互（useState、事件）的组件加 `'use client'`，其他保持 Server Component。

---

## ⚠️ 避坑指南

### 1. 忘记加 'use client'
如果在 Server Component 里用了 `useState`、`useEffect`、事件处理，会报错。需要在文件顶部加 `'use client'`。

### 2. 路由分组目录忘记建 layout.tsx
路由分组目录下必须有 `layout.tsx`，否则分组不生效。

### 3. Link vs router.push
- `<Link href="...">` — 用于 a 标签导航（SEO 友好）
- `router.push('...')` — 用于事件触发的编程式跳转
